Android memiliki banyak fitur dan API yang memudahkan pengembangan aplikasi, dan salah satunya adalah pengelolaan state screen. Ini adalah mekanisme yang digunakan untuk mengelola bagaimana aplikasi berinteraksi dengan layar dan respons terhadap perubahan dalam status layar, seperti rotasi, perubahan orientasi, atau ketika layar dihidupkan kembali setelah pengguna berinteraksi dengan aplikasi.

Apa Itu Screen State di Android?

Screen state atau status layar merujuk pada kondisi atau status tampilan layar yang dapat berubah tergantung pada interaksi pengguna atau sistem. Contoh perubahan status layar yang umum adalah:

  • Rotasi Layar: Pengguna memutar perangkat mereka dari posisi potret ke lanskap atau sebaliknya.
  • Pengaktifan/Penonaktifan Layar: Ketika layar perangkat dimatikan atau dinyalakan, seperti pada lock screen atau ketika aplikasi pergi ke background.
  • Multitasking: Aplikasi dapat berpindah antara foreground dan background, yang memengaruhi tampilan layar dan state UI.
  • Mode Tegak (Portrait) atau Lanskap (Landscape): Merubah orientasi layar yang mempengaruhi tata letak dan elemen UI.

Mekanisme yang menangani masalah ini di Android sangat penting agar aplikasi bisa beradaptasi dengan berbagai kondisi layar dan tetap memberikan pengalaman yang lancar dan konsisten bagi pengguna.

Mekanisme API Screen State di Android

Ada beberapa API dan pendekatan yang digunakan untuk menangani state layar di Android:

1. Activity Lifecycle

Android menggunakan lifecycle dari Activity untuk mengelola perubahan status layar. Setiap perubahan, seperti rotasi atau ketika aplikasi pindah ke background, akan memicu perubahan state dalam lifecycle, seperti onCreate(), onPause(), dan onResume().

  • onCreate(): Ketika activity pertama kali dibuka, Android akan memanggil onCreate(). Di sini, kamu bisa menyetel elemen UI dan inisialisasi lain.
  • onPause() & onResume(): Ketika aplikasi pindah ke background atau kembali ke foreground, metode ini akan dipanggil. Ini penting jika aplikasi harus menyimpan state (misalnya, nilai input) atau mengatur ulang UI sesuai keadaan layar.
  • onSaveInstanceState(): Ketika aplikasi mengalami perubahan (misalnya, rotasi layar), metode ini akan dipanggil untuk menyimpan state sementara, seperti data input pengguna.

2. Configuration Changes

Ketika terjadi perubahan orientasi layar atau pengaturan perangkat, Android secara default akan menghancurkan dan membuat ulang Activity. Hal ini bisa membuat aplikasi kehilangan state yang sedang berjalan (misalnya, data input atau scroll position).

Untuk menangani hal ini, Android memungkinkan kamu untuk mengatasi perubahan konfigurasi tanpa menghancurkan dan membuat ulang activity menggunakan dua pendekatan:

  • android:configChanges: Kamu bisa menambahkan tag ini di file AndroidManifest.xml untuk memberitahukan Android agar tidak menghancurkan Activity pada perubahan tertentu (seperti orientasi layar). Contoh:

    <activity android:name=".MainActivity"
          android:configChanges="orientation|keyboardHidden">
</activity>

  • onConfigurationChanged(): Ketika perubahan konfigurasi terjadi, metode ini akan dipanggil. Di sini, kamu bisa menyesuaikan UI aplikasi dengan cara manual (misalnya, memperbarui layout untuk orientasi baru).

3. Screen On/Off & Screen Lock State

Untuk memantau apakah layar perangkat sedang menyala atau terkunci, kamu bisa menggunakan Broadcast Receivers yang mendengarkan status perubahan layar, seperti:

  • Intent.ACTION_SCREEN_ON: Menerima notifikasi saat layar perangkat dihidupkan.
  • Intent.ACTION_SCREEN_OFF: Menerima notifikasi saat layar perangkat dimatikan.
  • Intent.ACTION_USER_PRESENT: Menerima notifikasi saat perangkat dibuka dari screen lock.

Dengan menggunakan broadcast ini, aplikasi bisa merespon ketika layar dihidupkan atau dimatikan. Misalnya, aplikasi bisa menghentikan atau melanjutkan pemutaran media berdasarkan status layar.

4. Keeping Screen On

Terkadang, kamu mungkin ingin memastikan layar tetap menyala selama aplikasi digunakan (misalnya, dalam aplikasi video atau game). Ini bisa dilakukan dengan menggunakan:

getWindow().addFlags(WindowManager.LayoutParams.FLAG_KEEP_SCREEN_ON);

Ini akan mencegah layar perangkat mati selama aplikasi berjalan. Sebaliknya, kamu bisa menggunakan:

getWindow().clearFlags(WindowManager.LayoutParams.FLAG_KEEP_SCREEN_ON);

untuk mengembalikan status layar ke normal.

Menjaga State UI pada Perubahan Layar

Salah satu masalah umum yang sering dihadapi pengembang adalah menjaga UI tetap konsisten saat terjadi perubahan layar, seperti rotasi perangkat. Pengguna mungkin tidak ingin kehilangan data yang telah mereka masukkan atau kehilangan posisi scroll yang sedang aktif. Beberapa cara yang bisa dilakukan untuk mengatasi ini:

  1. Menggunakan ViewModel dan LiveData: Ini adalah cara modern yang diperkenalkan oleh Android untuk menyimpan dan mengelola data UI secara lebih efisien saat terjadi perubahan konfigurasi.
  2. Menyimpan State di onSaveInstanceState(): Kamu bisa menyimpan data sementara dalam onSaveInstanceState() dan memulihkannya di onRestoreInstanceState() atau onCreate().
